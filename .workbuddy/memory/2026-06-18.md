# 2026-06-18 工作日志

## curl 及依赖库编译脚本

基于 Homebrew homebrew-core Formula 目录中的公式，为 mac_arm64 编写了 curl 及其全部依赖库的编译脚本。

### 依赖链分析

从 `Formula/c/curl.rb` 出发，完整依赖链：

```
curl (v8.20.0)
├── openssl@3  (v3.6.2) — SSL/TLS
├── brotli     (v1.2.0) — 压缩 (cmake)
├── zstd       (v1.5.7) — 压缩 (cmake)
│   ├── lz4    (v1.10.0)
│   └── xz     (v5.8.3)
├── libnghttp2 (v1.69.0) — HTTP/2
├── libnghttp3 (v1.16.0) — HTTP/3 (cmake)
├── libngtcp2  (v1.23.0) — QUIC
├── libssh2    (v1.11.1) — SSH
└── pkgconf    (v2.5.1) — 构建工具
```

### 编译顺序

1. openssl.sh → 2. xz.sh → 3. lz4.sh → 4. zstd.sh → 5. brotli.sh → 6. pkgconf.sh → 7. libssh2.sh → 8. libnghttp2.sh → 9. libnghttp3.sh → 10. libngtcp2.sh → 11. curl.sh

### 设计要点

- 每个脚本使用 `--prefix="$PREFIX"`，默认 PREFIX 为 `/Applications/EServer/Library/<库名>`
- 不使用单独 `--libdir` / `--includedir` 参数
- 依赖库路径通过环境变量引用（如 `OPENSSL_PREFIX`、`ZSTD_PREFIX`）
- curl.sh 通过 `PKG_CONFIG_PATH` 串联所有依赖库
- 使用 `-arch arm64` 和 `--host=arm64-apple-darwin` 确保 arm64 架构 (已按用户要求移除 -arch arm64 和 --host)

### GitHub Actions 工作流

在 `.github/workflows/` 下为每个库创建了独立的 YAML 文件，风格与项目中已有的 `build-*.yml` 一致：

| YAML | 触发 Tag |
|------|----------|
| `build-openssl.yml` | `openssl-*` |
| `build-xz.yml` | `xz-*` |
| `build-lz4.yml` | `lz4-*` |
| `build-zstd.yml` | `zstd-*` |
| `build-brotli.yml` | `brotli-*` |
| `build-pkgconf.yml` | `pkgconf-*` |
| `build-libssh2.yml` | `libssh2-*` |
| `build-libnghttp2.yml` | `libnghttp2-*` |
| `build-libnghttp3.yml` | `libnghttp3-*` |
| `build-libngtcp2.yml` | `libngtcp2-*` |
| `build-curl.yml` | `curl-*` |

每个 workflow 独立运行，通过 `Prepare EServer Library` 步骤从 `Library/` 目录加载预置依赖，构建完成后使用 `softprops/action-gh-release@v2` 打包发布。
